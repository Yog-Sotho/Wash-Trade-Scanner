# Wash Trade Detection System

A professional, production-grade tool for auditing blockchains for wash trading and fake volume generated by bots.

## Features

- **Multi-Chain Support**: 20+ blockchains including Ethereum, BSC, Polygon, Arbitrum, Optimism, Base, Avalanche, Fantom, Celo, Gnosis, Moonbeam, Aurora, Harmony, Cronos, Metis, Boba, zkSync Era, Polygon zkEVM, Linea, Scroll, Mantle, Kava, Klaytn, Solana, and NEAR.
- **Multi-DEX Support**: Uniswap V2/V3, PancakeSwap, Sushiswap, Curve, Balancer, TraderJoe, QuickSwap, Velodrome, Aerodrome, SyncSwap, and more.
- **Heuristic Detection**: High-signal rules with near-zero false positives:
  - Self-trading detection
  - Circular trading pattern analysis
  - High-frequency bot identification
  - Volume anomaly detection
  - Entity clustering based on funding links
- **Machine Learning**: Isolation Forest for unsupervised anomaly detection, trainable on historical data.
- **Real-time Monitoring**: WebSocket-based event listeners for live swap detection.
- **Comprehensive Auditing**: Full historical analysis with detailed reporting.

## Installation

### Using Docker (Recommended)

```bash
# Clone the repository
git clone https://github.com/yourusername/wash-trade-detector.git
cd wash-trade-detector

# Copy environment variables
cp .env.example .env
# Edit .env with your RPC URLs and API keys

# Build and start services
docker-compose up -d

# Run an audit
docker-compose exec app python scripts/run_audit.py --chain-id 1 --pool 0x...
```

### Manual Installation

```bash

python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

pip install -r requirements.txt

from core.storage import Storage

async def main():
    storage = Storage()
    await storage.initialize()
    
    ingestor = MultiChainIngestor(storage)
    await ingestor.initialize()

    await asyncio.Event().wait()

asyncio.run(main())
```

## Detection Methodology

### Heuristics

1. **Self-Trading**: Sender == Recipient address.
2. **Circular Trading**: Strongly Connected Components (SCCs) in the trader graph.
3. **High-Frequency Bot**: Consistent trade sizes with sub-minute intervals.
4. **Volume Anomaly**: Statistical outliers in trade volume (z-score > 3).
5. **Wash Clusters**: Trades between addresses with shared funding sources.

### Machine Learning

The system uses an Isolation Forest algorithm trained on features including:
- Trade volume and slippage
- Trader activity frequency
- Pair-wise trade patterns
- Pool-level aggregate statistics
- Gas price and timing information
